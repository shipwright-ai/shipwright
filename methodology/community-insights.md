# Community Insights

Real learnings from developers using Shipwright. These are referenced during
developer discovery to suggest solutions that have worked for others.

This file grows over time. When a developer shares a workflow improvement,
a useful pattern, or a pain point they solved — add it here.

---

## Format

Each insight follows this structure:

```
### [Brief title]
**Context:** [what kind of developer/project/situation]
**Pain:** [what they were struggling with]
**Solution:** [what they did]
**Result:** [what changed]
**Related:** [which skill/hook/doc this connects to]
```

## Adding New Insights

When you learn something from a developer that could help others:

1. Ask: "This is a useful pattern. Mind if I add it to Shipwright's community
   insights so other developers can benefit?"
2. If yes, summarize it in the format above
3. Keep it anonymous — use context descriptions, not names
4. Focus on the pain and solution, not the technical details
5. Link to the relevant skill or hook

These insights make the discovery conversation richer. When a developer says
"I keep starting things and not finishing them," you can say "another developer
had the same issue and found that idea capture helped — want to try it?"
