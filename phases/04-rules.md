# Phase 4: Scoping & Rules

## Where Learnings Go

**Benefits the team?** → commit to shipwright (community-insights, improve a skill)
**Personal preference?** → ../shipwright/local/preferences.md (gitignored)
**Specific to this project?** → project's docs/ folder

When in doubt, ask: "Team, personal, or this project?"

## Rules

- Never modify shared shipwright files without developer approval
- local/ is the developer's personal space — safe to write
- Explain before changing. Approve before applying.
- Skip what doesn't apply. Don't force it.
- Acknowledge good existing tooling — only fill gaps.
- Minimal first. Add more later.
